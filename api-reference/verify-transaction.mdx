api: 'GET https://api.rollapay.com/transaction/verify/{reference}'

---

After a customer makes a payment, the next step is to verify that the payment was successful before delivering value to them. This guide walks you through verifying transactions.

## Why Verify?

When a customer completes a payment on your checkout page and is redirected back to your site, you need to confirm that the payment was actually successful. This is important because:

1. **Security**: Client-side data can be manipulated. A malicious user could fake a successful callback.
2. **Reliability**: Network issues might cause webhooks to fail or be delayed.
3. **Accuracy**: Get the exact amount paid and confirm it matches the expected amount.

<Warning>
  Never deliver value based solely on the redirect URL or client-side data. Always verify on your server.
</Warning>

## How It Works

When a transaction is completed, the transaction reference is returned as a **query parameter** to your `callbackUrl`:

```
https://yoursite.com/callback?trxref=TXN_gGbEEvK25eVgmgrf&reference=TXN_gGbEEvK25eVgmgrf
```

You'll extract this reference and send it to the verify endpoint from your server.

## Understanding the Response

<Info>
  **Important**: The API response has a `success` field indicating if the API call was successful. This is **not** the status of the transaction.

  The transaction status is in the `data` object: `response.data.status`
</Info>

| Field | What It Means |
|-------|---------------|
| `response.success` | Whether the API request succeeded |
| `response.data.status` | The actual transaction status (`successful`, `failed`, `pending`, etc.) |

## Authentication

All requests must include your secret API key in the Authorization header:

```
Authorization: Bearer sk_live_your_secret_key
```

<Note>
  Use your **secret key**, not your public key. The organization is automatically determined from your API key.
</Note>

## Path Parameters

<ParamField path="reference" type="string" required>
  The unique transaction reference returned when the transaction was initiated (e.g., `TXN_gGbEEvK25eVgmgrf`)
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rollapay.com/transaction/verify/TXN_gGbEEvK25eVgmgrf" \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```javascript Node.js
const reference = 'TXN_gGbEEvK25eVgmgrf';

const response = await fetch(`https://api.rollapay.com/transaction/verify/${reference}`, {
  headers: {
    'Authorization': 'Bearer sk_live_your_secret_key',
  },
});

const result = await response.json();

if (result.success && result.data.status === 'successful') {
  // Payment was successful
  // Deliver value to customer
  console.log('Payment successful!');
  console.log('Amount paid:', result.data.amount);
  console.log('Paid with:', result.data.payment_source?.brand);
} else {
  // Payment failed or is pending
  console.log('Payment not successful:', result.data.status);
}
```

```python Python
import requests

reference = 'TXN_gGbEEvK25eVgmgrf'

response = requests.get(
    f'https://api.rollapay.com/transaction/verify/{reference}',
    headers={
        'Authorization': 'Bearer sk_live_your_secret_key',
    }
)

result = response.json()

if result['success'] and result['data']['status'] == 'successful':
    # Payment was successful
    print('Payment successful!')
    print(f"Amount paid: {result['data']['amount']}")
    if result['data']['payment_source']:
        print(f"Paid with: {result['data']['payment_source']['brand']}")
else:
    # Payment failed or is pending
    print(f"Payment not successful: {result['data']['status']}")
```

```php PHP
<?php
$reference = 'TXN_gGbEEvK25eVgmgrf';

$ch = curl_init();

    // Payment failed or is pending

}
?>
```

## Transaction Statuses

| Status | Description | Action |
|--------|-------------|--------|
| `pending` | Transaction created but not yet processed | Wait for customer to complete payment |
| `requires_action` | Additional action required (3DS, PIN, OTP) | Customer has not completed authentication |
| `processing` | Transaction is being processed | Wait and poll again |
| `successful` | Payment completed successfully | **Deliver value to customer** |
| `failed` | Transaction failed | Do not deliver value |

## Step-by-Step Verification

Here's how to properly verify a transaction:

<Steps>
  <Step title="Receive the callback">
    When the customer is redirected to your `callbackUrl`, extract the reference from the query parameters:
    ```javascript
    const urlParams = new URLSearchParams(window.location.search);
    const reference = urlParams.get('reference');
    ```
  </Step>
  <Step title="Send reference to your server">
    Send the reference to your backend. Never verify from the client side.
    ```javascript
    const response = await fetch('/api/verify-payment', {
      method: 'POST',
      body: JSON.stringify({ reference }),
    });
    ```
  </Step>
  <Step title="Verify on your server">
    Call the Rolla verify endpoint with your secret key:
    ```javascript
    const result = await fetch(`https://api.rollapay.com/transaction/verify/${reference}`, {
      headers: { 'Authorization': `Bearer ${process.env.ROLLA_SECRET_KEY}` },
    }).then(res => res.json());
    ```
  </Step>
  <Step title="Check the transaction status">
    Confirm the transaction was successful **and** the amount matches:
    ```javascript
    if (result.data.status === 'successful') {
      // Verify the amount matches what you expected
      if (result.data.amount === expectedAmount) {
        // Deliver value to customer
      }
    }
    ```
  </Step>
</Steps>
