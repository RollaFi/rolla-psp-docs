## Overview

Webhooks allow you to receive real-time notifications when events occur in your Rolla account. When an event happens, we send an HTTP POST request to your configured webhook URL.

## Setting Up Webhooks

Configure your webhook URL in the [Rolla Dashboard](https://merchant.rollapay.com) under **Settings > API Keys & Webhooks**.

You can configure separate webhook URLs for test and live modes.

## Webhook Events

| Event                | Description                               |
| -------------------- | ----------------------------------------- |
| `payment.pending`    | Payment has been initiated and is pending |
| `payment.processing` | Payment is being processed                |
| `payment.success`    | Payment completed successfully            |
| `payment.failed`     | Payment failed                            |
| `refund.completed`   | Refund was processed                      |
| `dispute.created`    | A dispute has been opened                 |

## Webhook Headers

Each webhook request includes the following headers:

| Header               | Description                                  |
| -------------------- | -------------------------------------------- |
| `Content-Type`       | Always `application/json`                    |
| `X-Rolla-Signature`  | HMAC-SHA256 signature for verification       |
| `X-Rolla-Event`      | The event type (e.g., `payment.success`)     |
| `X-Rolla-Timestamp`  | ISO 8601 timestamp when the webhook was sent |
| `X-Rolla-Webhook-Id` | Unique webhook delivery ID                   |

## Verifying Webhooks

<Warning>
  Always verify webhook signatures to ensure requests are from Rolla.
</Warning>

Verify the signature using your webhook secret:

```javascript Node.js
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

// Express.js example
app.post('/webhook', express.json(), (req, res) => {
  const signature = req.headers['x-rolla-signature'];
  const webhookSecret = process.env.ROLLA_WEBHOOK_SECRET;

  if (!verifyWebhook(req.body, signature, webhookSecret)) {
    return res.status(401).send('Invalid signature');
  }

  const { event, data } = req.body;

  switch (event) {
    case 'payment.success':
      // Handle successful payment
      console.log('Payment successful:', data.reference);
      break;
    case 'payment.failed':
      // Handle failed payment
      console.log('Payment failed:', data.reference, data.failureReason);
      break;
  }

  res.status(200).send('OK');
});
```

```python Python
import hmac
import hashlib
import json

def verify_webhook(payload, signature, secret):
    expected_signature = hmac.new(
        secret.encode('utf-8'),
        json.dumps(payload).encode('utf-8'),
        hashlib.sha256
    ).hexdigest()

    return hmac.compare_digest(signature, expected_signature)

# Flask example
@app.route('/webhook', methods=['POST'])
def webhook():
    signature = request.headers.get('X-Rolla-Signature')
    webhook_secret = os.environ.get('ROLLA_WEBHOOK_SECRET')

    if not verify_webhook(request.json, signature, webhook_secret):
        return 'Invalid signature', 401

    event = request.json.get('event')
    data = request.json.get('data')

    if event == 'payment.success':
        # Handle successful payment
        print(f"Payment successful: {data['reference']}")
    elif event == 'payment.failed':
        # Handle failed payment
        print(f"Payment failed: {data['reference']}")

    return 'OK', 200
```

## Responding to Webhooks

Your endpoint should return a `2xx` status code within 30 seconds. If we don't receive a successful response, we'll retry the webhook:

| Attempt   | Delay      |
| --------- | ---------- |
| 1st retry | 1 minute   |
| 2nd retry | 5 minutes  |
| 3rd retry | 15 minutes |
| 4th retry | 1 hour     |
| 5th retry | 2 hours    |

After 5 failed attempts, the webhook will be marked as failed.

## Best Practices

<AccordionGroup>
  <Accordion title="Return 200 quickly">
    Process webhooks asynchronously and return a 200 status immediately. Handle
    business logic in a background job.
  </Accordion>
  <Accordion title="Handle duplicates">
    Webhooks may be delivered more than once. Use the `webhookId` field to
    deduplicate events.
  </Accordion>
  <Accordion title="Verify signatures">
    Always verify the `X-Rolla-Signature` header before processing the event.
  </Accordion>
  <Accordion title="Use HTTPS">
    Always use HTTPS for your webhook endpoint to ensure data is encrypted in
    transit.
  </Accordion>
</AccordionGroup>
